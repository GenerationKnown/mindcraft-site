DevelopersBack to top

Before getting into the nitty griddy of developer jargon, let's just cover what you need to know. First of all, we are using minecraft, a game developed by Mojang. It's an open world that is extremely easy to modify and create with. Our modding is supported by bukkit, a wrapper for the minecraft server. It allows us to add our mods, which are Denizens, World Edit, and World Guard.

Denizens is a mod that allows moderators of the server to add NPCs and scripts to the world. This mod is used to make sentries and quizzers, the actual educational parts of learning landscape.

World Edit is a mod that allows developers to edit the world using in game code lines typed into chat. You can learn more about World Edit below.

World guard is a mod that can set areas and permissions. These permissions include block permissions, destruction permissions, and other permissions in the area.

Learning landscape combines many component technologies.

TeachersBack to top

Learning Landscape is a server in MineCraft dedicated to educational gameplay. There are locational nodes that each indicate a particular subject or topic. Each of these nodes has sentries, or non-playable characters (NPCs) that are capable of asking questions.

Although there are limited nodes, we have created tutorials so that teachers from all over the world may add their own material and questions. It is our hope that Learning Landscape will become a visual, playable and engaging knowledge map.

How do I submit new material?

How to format a text file so it can be read by the Terrain Generator:

  1. Step 1: Find the subject you want to map on Khan Academy.
  2. Step 2: Get the URL of that subject. It will look something like this: https://www.khanacademy.org/math/e/addition_2
  3. Step 3: Use that URL to create the formatted node like so:

The file format has 5 lines per subject.

DisplayName:
This is the title of the subject in proper English. The player sees this. If you go to the URL, this is the visible title of the page. (In this case: "2-digit addition")
Filename:
This is the title of the subject in a language the computer can read better. Computers don't like spaces, they like underscores. You can find your filename at the end of the URL. (In this case: "addition_2")
URL:
The KhanAcademy URL in its entirety.
PreReqs:
The pre-requisite subjects that the user must pass to access this subject. Each pre-requisite subject is referenced by FileName, not DisplayName. If there are no pre-reqs, write N/A instead. (In this case: "number_line", "addition_1", and "representing_numbers")
(vPos,hPos):
The actual position of the subject on the entire knowledge map. *If your subject is not Math, write (0,0) in this field.
*Be certain to include all colons, spaces, quotes, and other symbols as seen in the examples below*

Create a 5-line node for each subject and save it as a text file to have the Terrain Generator read it!

EXAMPLE:

DisplayName: "Representing numbers"
FileName: "representing_numbers"
URL: "https://www.khanacademy.org/math/e/representing_numbers"
PreReqs: [N/A]
(vPos,hPos): (3,0)
