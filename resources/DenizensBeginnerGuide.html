Introduction:

        “Denizen, powered by Citizens2, is an extensive add-on pack that allows for a scriptable Minecraft experience, including enhanced  NPCs, custom commands, items, world events and many other features. Inspired by MUDs, features and NPCs powered by Denizen have a unique RPG feel unmatched by any other CraftBukkit solution.”

Source: Denizens 0.8 Handbook

        Denizens can be used for a variety of purposes, such as quests, learning tools, interactive dungeons, and much more! With the combination of Denizens, and the creativity of players, endless possibilities are available. This guide will discuss the basics of creating and using Citizens-based NPCs, and Denizens-powered scripts.

Setting up a Server

Setting up the server is the most important part of using Denizens, as the server is the platform used to run all the plugins and scripts used. For this tutorial, a server package has been compiled for ease of use. For a full guide on how to set up a Bukkit server, visit this site. If you already have a bukkit server ready, simply install Citizens and Denizens.

Using the server

  1. Unrar the server package and extract to a folder where it will be easily accessible.
  2. Run the program “BukkitGUI”. Update as necessary.
  3. Go to the “Superstart” page and be sure the JAR file is set to (FolderYouExtractedTo)\craftbukkit.jar.
  4. In the topright corner, press “Get current version #”. If the current version of craftbukkit is outdated, press “Download Latest Recommended Build”.
  1. OPTIONAL: If you do not own a Minecraft account, or do not have internet access, you may set the server to offline mode by pressing the Server Options tab, then set online-mode to false.
  1. When you are ready, start the server. Connect by adding a server in your Minecraft multiplayer screen, and setting the IP to localhost.
  1. Modify any other options as needed.

Creating Your First NPC

Now that your server is up and ready, enter the server and find a nice place for a new NPC.

(A full guide to NPC commands can be found at the Citizens Wiki)

Getting Familiar with NPCs

  1. Create an NPC by typing “/npc create ‘name’”
  1. Do not include the quotes, and replace ‘name’ with the name of the NPC.
  2. You may notice that the NPC does not have the typical “Steve” skin. This is because NPCs adopt the skin of the player it is named after.
  1. Now, let’s create a non-human using the --type modifier. Make a new NPC using “/npc create ‘name’ --type Enderman”.
  1. This will create an NPC that has an Enderman form. Other mobs can be used such as pigs or villagers.
  1. Perhaps we want to change the first NPC we created, somehow. There are several ways to select it.
  1. 1. Right click the NPC with a stick.
  2. 2. Face the NPC and type “/npc select”.
  3. 3. Use “/npc list” to view all NPCs. Use “/npc select ‘name or ID’” to select it.
  1. Try teleporting the NPC to you by using “/npc tphere”. You may also teleport to the NPC by using “/npc tp”.
  2. We’ll make the NPC a bit more lifelike by making it face you when a player is near. Use “/npc lookclose” to create this effect.
  3. To discard an unwanted NPC, select it and use “/npc remove”
  4. Visit the Citizens Commands page and try out all the commands to become familiar with them.

Parts of a Denizen Script

        A denizen script consists of 4 main parts. The name, the type, the requirements, and the steps. The steps is where the actual script is placed. Type, requirements, and steps are all a subgroup of the name, and each of them have subgroups of their own. For example, a script may have this format:

Examplescript:                                     ←NAME

  Type: Interact                                   ←TYPE

  Requirements:                                    ←REQUIREMENTS

    Mode: all

    List:

    - -Script Finished ‘Script:ExampleScript’[a]     

  Steps:                                           ←STEPS (Script)

    1:

      Click Trigger:

        Script:

        - CHAT “Hello! This is a chat command!”

        - ANNOUNCE “And this is an announce command! The whole server can

hear this!”

        - FINISH

Your First Denizen

        

        Denizens are NPCs that run a predefined script. This tutorial will show you step by step how to create a basic script and demonstrate the various triggers.

  1. Open the server folder and enter the “plugins” folder. If a denizens folder does not exist, make sure both Citizens and Denizens are installed and that you have run the server.
  2. Open the Denizen folder and create a new folder named “Scripts”.
  3. Create a document and save it to the Scripts folder as a .yml.
  1. Be sure to give it a unique name! (Not “Script.yml”)
  1. First, create a new script by giving it a name. The first line should be as follows:
    ScriptName:
  2. Set the script as an “Interact” script. This makes the script able to use triggers to run.
  1. Note! YML is picky about indentations! Use only spaces! I recommend using 2 spaces for an indent. Your script should look as follows:
    ScriptName:
     Type: Interact
  1. Set the requirements. A list of requirements can be found at the Denizens Wiki.
  1. In this case, we will use the TIME requirement.
  2. Set mode to None if you don’t want any requirements.
    ScriptName:
     Type: Interact
     Requirements:
       Mode: All
       List:
       - TIME Day
  1. Add the Steps. You may add as many steps as needed. On the first interaction, the NPC will start at Step 1, and stay there until a ZAP is used to take us to step 2. This will be discussed in future tutorials.
  1. Note: Steps is the same indentation as Type and Requirement.

 Requirements:

    Mode: All

  Steps:

    1:
   2:

Moderate Scripting

        This next tutorial will display a few more advanced commands and requirements, as well as priorities in assignment scripts. FLAG and FLAGGED are tools that will be used in this tutorial to check for specific player-made requirements to be met. Different actions will occur based on the current flag value.

A Bit of Information:

The FLAG command is used to temporarily store a value. Flags can be assigned to the player, to the NPC, or globally.

Usage:

flag ({player}|npc|global) [name([#])](:action)[:value] (duration:#)

(Taken from the Denizens 0.8 Handbook)

The FLAGGED requirement checks if a flag is present, and matches a defined boolean, string, or integer.

Usage:

- (-)flagged [player|npc|global] [name([#])](:value)

18. Finally, we’ll make an assignment script and set the priorities.

  1. Priorities go in order of descending rank. For example, it will check the requirements of a priority 10 script before a priority 9 script.
  2. In this case, the scripts will be as follows:
  1. It will try to run the quest completed script. If the player meets the requirements (already flagged as a town hero), the script will run, if not, it will check the next script.
  2. The next script will be the checking for items. First it will run “HasItems” to check if the player has 1: Already completed the acceptance script, and 2: Has all the items. If the player is missing either, it will move onto the next script.
  3. It will check the requirements of “DoesNotHaveItems”. This way, if the player has already done the first script, but has NOT met the item requirements from the last script, it will run.
  1. Since there are only two possibilities (Either has items or does not), then if the player fails both of these, the player must not have met the other requirement (Done with the first script).
  1. It will check the “moderatescript” requirements.
  1. Honestly, the requirements on this are unnecessary. Since it has already checked that they have not done moderate script, and they already failed the test for the TownHero flag, it MUST be this script.

QuestAssignment:

  Type: Assignment
 Interact Scripts:

  - 4 QuestIsAlreadyDone

  - 3 HasItems

  - 2 DoesNotHaveItems

  - 1 ModerateScript

21. Test that the script works.

Hey, Listen!

Introduction:

Listeners can be used to check if events in the world have been completed. This is useful for tasks such as gathering items, killing enemies, or even creating custom item drops.

Types of listeners:

- KILL
- BLOCK
- ITEM
- TRAVEL
- ITEMDROP

See the Denizens 0.8 Guide for more information on each.

Tasks are used to run a chunk of script. These are helpful for IF statements, to run when Listener quests are done, or even just when you don’t want to retype the same piece of script over and over.

Gather 10,000 grass!

Not really, but with listeners, that quest would be possible. However, for this example, I’ll be using a KILL listener to check if a player has killed a certain number of creepers.

1. Make a new script and name it as needed. Make it an interact script (review previous tutorials if needed).

2. Write a few lines of script for the introduction of the quest.

KillCreepersStart:

  Type: Interact
 Requirements:

    Mode: All

    List:
   - -script finished ‘script:KillCreepersStart’
 Steps:
   1:
     Chat Trigger:

        1:
         Trigger: /Regex:Hi|Hello|Hey/
         Script:
         - CHAT “<PLAYER.NAME>! You have to help me! There’s been a sudden boom in Creeper population!

          - WAIT 2
         - CHAT “I need you to help control the crowd, and kill 10 creepers for me!”
         - CHAT “When you do, I’ll give you a great reward!”

3. Now, we’ll create a TASK script to run when the player completes the quest. To do this, name the script, define it as Type: Task, and then add the script line. There are no steps or requirements.

CreepersDead:

  Type: Task

  Script:

  - NARRATE “You have killed 10 creepers! Return to Farmer to claim your reward!”

  - FLAG CreepersKilled:10

4. Add the LISTEN command to the quest start script. This will start the “quest”. The syntax is as follows for a kill listener:

  1. listen kill [type:NPC|PLAYER|GROUP|ENTITY] [target:entity_type] (qty:##) (region:region_id)

5. Add the ‘ID:ListenerName’ argument to give the listener a unique ID.

6. Add the ‘script:scriptname’ argument to make it run the quest completion script when it’s

done. (In this case, CreepersDead).

The Gambler:

Introduction:

Sometimes, you don’t want to write the same chunk of script over and over. For this reason, we use Tasks. IF statements can also check for various requirements, without needing to write a new script with new requirements.

Tasks are used as scripts that are run only by scripted cues. They cannot be triggered by any normal trigger, but will only run when another script calls it. It is useful for keeping groups of script together, or for when you want multiple things to run the same script, without having to re-type that portion of script each time.

Instructions:

1. Create a new interact script and assignment script.

2. Make a RANDOM command.

  1. RANDOM will choose randomly from the next “x” lines
  2. Ex. ‘RANDOM 5’ will choose from the next 5 lines.

3. Create a few lines for the number of RANDOMs you’d like. For each, put a RUNTASK command.

  1. Syntax: RANDOM scriptname

4. Create a new script for each unique runtask. This is done much like any other script:

  1. The name line
  2. The type line (Type: Task)
  3. Instead of having requirements or scripts, this line will simply be Script:
  4. Any commands beneath the Script line.

Scriptception (Putting it All Together) :

This tutorial will show how to create a series of repeatable quests using tasks and flags.

Note: IF statements can be used for a LOT more. This tutorial will only discuss comparing flags, values, and MATCHING arguments.

1. Create a new interact script. (Review previous tutorials if needed).

2. Make a few steps using chat triggers that describe the quests.

  1. Do not include requirements yet. (use Requirements: None to specify no requirements).
  2. Steps do not need to be numbered, you can give them descriptive names!

3. Make a TASK script that will start the listener for each quest.

  1. In each, set a flag ‘QuestInProgress’ to a descriptive name of the quest.

4. At the end of each description, make an IF statement to check if the QuestInProgress already has a value. If not, run the quest TASK script to start it.

  1. Look below for the IF statement instructions


IF Statements:


Make your own simple IF statement using this table!

== checks if the flag is equal to a value
!= checks if the flag is NOT equal to a value
> checks is the flag is greater than a value
< checks if the flag is less than a value
>= checks if the flag is greater than or equal to a value
<= checks if a flag is less than or equal to a value
matches checks if a value is the same as another.

  1. Example: IF “Entez” matches player narrate “Entez is a player!”

contains checks if a list contains a value.

  1. Example: if 'gold|iron|diamond' contains <player.item_in_hand.material> narrate “You are holding a rare item!”
  2. If <PLAYER.NAME> contains ‘ent’ narrate “You are probably Entez!”

Steps:

  1. Replace the NAME in <FLAG.P:NAME || FALLBACK> to the name of your flag.
  2. Replace Fallback with the value you want to return when the flag does not exist.
  1. I.E. “Null” or “None”
  1. Use the operator of choice.
  2. Replace “Value” with either a string, integer, boolean, OR another flag.
  1. Another flag can be used to check if two flags have the same (or different) value.
  1. Place the full command to run. If you want to run one line, you can put a single command. Otherwise, you’ll have to run a task.
  1. If <FLAG.P:REPUTATION> >= 100 CHAT “<PLAYER.NAME>, the hero has returned!”
  2. IF <FLAG.P:REPUTATION> <= 5 RUNTASK ‘SCRIPT:RUNOUTOFTOWN’
  1. If you want something to run when the IF fails the check, use ELSE, followed by a command.

5. Create a TASK script for each quest and reward.

6. Using an IF statement, check if the player currently has a quest in progress. If so, tell them to finish that quest before taking a new one. If not, allow them to accept the quest by running the TASK for the quest.

7. Create a quest for teach task script you’ve created (not the reward tasks, yet) using listeners. Use the reward script you’ve created as the ‘Script:scriptname’ argument in the listener.

  1. Remember to assign a unique ID for each listener!

8. In the reward script, create a few lines that give the player something, and reset the flag for the current quest in progress to none. This will allow the player to accept a new quest once they have completed one.

Troubleshooting

Q: My NPC is not doing anything when I talk to it!

A:

  1. Be sure that the NPC accepted the assignment correctly. (Select it and use /npc assignment).
  2. Check that the necessary triggers are enabled. (If chat trigger is needed, do /npc trigger to check if it is enabled. If not, do /npc trigger chat -t to toggle it)
  3. Check that the script loaded correctly. (On a reboot or reload, check your server console for any errors).
  4. Check your YAML for syntax errors here. (Many times a tab will break the script. Be sure to only use spaces)
  5. If all else fails, ask on the Denizens Dev Channel.

Q: I used multiple steps, and whenever I talk to the NPC, it skips to step two!

A: Put a ZAP back to step 1 at the end of the script. This will make it start over whenever you talk to them again.

Q: Can I make a quest repeatable after a certain period of time?

A: You can use flags with a duration to achieve this effect.

  1. ex. FLAG TemporaryFlag duration:30m will keep it flagged for 30 minutes, and remove it afterwards.
