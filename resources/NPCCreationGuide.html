NPC Creation Guide for Learning Landscapes

By Ian Kane[a]

In order to use this guide, you need to know basic Denizens scripting, knowledge of Remote Access, and access to Minecraft. You should also have some background in bugfixing.

First of all, you need to Remote Access the server. Open the start menu and search “Remote Desktop” and open up the program. Below is the information you need to log in:

Server info: 129.21.142.118

Username: kinect

password: Milky

After you do this, accept the warning and you’ll be accessing the server.

== IN REMOTE

Next, you’ll need your script to use. You should have this script ready before remote accessing, or at least ready for debugging. Put the script into the Denizens/Scripts folder, then log into the server on your machine via Minecraft.

-- IN MINECRAFT

Type

/denizen reload -a

to reload the denizens mod, giving you access to your new script.

== IN REMOTE

make sure it compiled correctly. If you get errors, you must debug your script, which will take your own debugging knowledge. if you think you have fixed it

-- IN MINECRAFT

Reload the denizens mod and then check if it’s loading correctly. Keep going back and forth between compiling the program and checking the server window to see if there are any errors.

After Denizens properly reloads and the script and everything is working, you need to assign the script to the npc. First of all, place an npc by typing

/npc create NAME_OF_NPC                        

//replace NAME_OF_NPC With your npc name.

This should create the NPC right where you are standing. if you just created an NPC, it will automatically be the selected NPC. If you want to select a new NPC, point your cursor at him and type

/npc select

and your NPC will become selected one. Now, to put your script on the NPC, type

/npc assignment --set ASSIGNMENTNAME

                        REPLACE THIS WITH YOUR SCRIPT’S ASSIGNMENT NAME

And the script should be stuck onto your NPC.

ISSUES THAT WILL HAPPEN:

Proximity trigger

if you have a Proximity trigger, make sure you enable it. First of all, select the NPC you need to have the trigger working for. Next, type in

/npc trigger

To check the triggers it has enabled and disabled. You should see PROXIMITY there with Disabled next to it. to change this, type

/npc trigger proximity -t

to toggle it on. If you want it off, type that line again to change that. To make sure the trigger dosn’t go off over and over, make the cooldown longer. This can be done by typing

/npc trigger proximity --cooldown FLOAT

FLOAT can be any number with 1 decimal place, make sure it has something in the decimal place. 10.0 is the default value. If you want to change the radius the proximity is checking, type

/npc trigger proximity --radius INT

Where INT is an integer value with no decimal points.
